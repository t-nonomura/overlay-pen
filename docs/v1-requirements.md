# V1 Requirements

## Product Goal

ユーザーが現在見ている Android 画面の上に、別アプリへ移動せずに素早く落書きやマーキングを残せること。

## Release Model

- V1 のファーストリリースは無料版のみとする
- V1 では Google Play Billing を使ったアプリ内課金や機能解除を行わない
- 将来の有料版候補機能は内部実装される場合があるが、V1 公開版ではユーザー導線を持たない
- 有料版を正式検討する条件は、無料版の市場需要と継続利用が確認できた後とする

## Free Version Definition

無料版の完成条件は「オーバーレイ描画の基幹価値を、審査と運用に耐える最小スコープで安定提供できること」とする。

### Included In First Release

- フローティングボタンによる開始
- 全画面透明キャンバスでの描画
- ペン、消しゴム、アンドゥ、全消し
- 色、太さ、透明度、ペン種類の変更
- 無料版の基本色は黒、赤、青、黄の 4 色とする
- `Keep` による passive annotation 維持
- `Hide` による compact `Tools` モード
- 通知からの再開、全消し、停止

### Excluded From First Release

- アプリ内課金
- 購入導線
- 有料プランの案内
- 無料版 UI から見える premium 告知
- 有料版前提のアカウント連携や entitlement 管理

## Target User

- オンライン会議や対面説明で画面を指し示したい人
- 手順メモや一時的な注釈を残したい人
- スクリーンショット保存までは不要だが、いま見ている画面に補助線や印を置きたい人

## Primary User Flow

1. アプリを起動する
2. オーバーレイ権限の必要性を理解し、設定画面で許可する
3. フローティングボタンを表示したまま他アプリを使う
4. フローティングボタンをタップして落書きモードへ入る
5. 画面上に描画し、消しゴムやアンドゥやペン設定を使う
6. 落書きモードを終了する
7. 描画は残したまま、下のアプリ操作へ戻る
8. オーバーレイツール自体を終了すると描画も消える

## Functional Requirements

### FR-01 Overlay Launcher

- アプリは起動後にフローティングボタンを表示できる
- フローティングボタンは画面上でドラッグ移動できる
- フローティングボタンは他アプリ利用中でも表示される

### FR-02 Enter Drawing Mode

- フローティングボタンをタップすると全画面の透明キャンバスを開く
- 描画中はツールバーを表示する
- 描画中は下のアプリ操作をブロックし、誤タップを防ぐ

### FR-03 Drawing Tools

- 少なくとも以下を提供する
  - ペン
  - 消しゴム
  - アンドゥ
  - ペン太さ変更
  - ペン色変更
  - 透明度変更
  - ペン種類変更
- V1 のペン種類は次の 3 種までに制限する
  - Solid
  - Highlighter
  - Dashed

### FR-04 Exit Drawing Mode

- ユーザーが落書きモードを終了すると、下のアプリ操作へ戻る
- 終了時点の描画は保持され、オーバーレイとして見え続ける
- フローティングボタンは引き続き表示される
- V1 では下位アプリの操作性を優先し、Passive Annotation 用 overlay window alpha は `0.78` に固定する

### FR-05 End Overlay Session

- ユーザーがオーバーレイツールを終了すると、表示中の描画データはすべて破棄される
- 次回起動時に前回の描画は復元しない

### FR-06 Settings and State

- 描画中の設定変更は即時に反映される
- 同一セッション中は最後に使ったペン設定を保持する
- 画面回転時もセッション継続中であれば描画状態を保持する

### FR-07 Premium Readiness

- V1 のコードベースは、将来的に機能解放型の有料版へ拡張可能な構造を持てると望ましい
- ただし V1 公開版では premium 機能への表示導線、購入導線、誤操作での露出を作らない
- premium 候補機能を内部実装する場合は、無料版機能から明確に分離し、無効時の影響がないことを確認する

## Non-Functional Requirements

### NFR-01 Performance

- ミッドレンジ端末で通常描画時に目立つカクつきがない
- 単一ストロークの入力から描画反映までが体感上遅れない
- 30 分連続利用でサービスが異常終了しない

### NFR-02 Reliability

- Overlay 権限未許可時は安全に機能停止し、再許可導線を表示する
- Foreground Service の通知は常時表示し、ユーザーが状態を理解できる
- クラッシュ時に下のアプリの操作不能状態を残さない

### NFR-03 Privacy and Security

- 画面内容の取得、録画、OCR は V1 で行わない
- 画像保存や外部共有は V1 の対象外とする
- 収集データはクラッシュログなど最小限に留める

### NFR-04 Usability

- 初回導線で「なぜオーバーレイ権限が必要か」を説明する
- 主要操作は片手で到達できるサイズと位置にする
- 誤って終了しても影響が限定的になるよう、終了時は明示操作にする

### NFR-05 Release Safety

- 無料版では未解放機能が UI 上に露出しない
- 未解放機能を含む場合でも、クラッシュ、描画性能、メモリ使用量に無料版の主導線へ悪影響を与えない
- 将来の課金導入を見越した設計をしても、V1 のコード複雑性と検証負荷を過度に増やさない

## Current Known Gaps Against V1

- 権限説明は最小 UI で、理由説明や初回オンボーディングは今後改善余地がある
- 実機での 30 分連続利用や OEM ごとの差分検証は未完了
- 金融系や認証系など overlay 非表示アプリでの挙動は未整理
- Google Play 前提の説明文と審査対応方針は未確定

## Acceptance Criteria

1. ユーザーが権限を許可すると、他アプリ上にフローティングボタンを表示できる
2. フローティングボタンから描画モードへ入り、指で連続描画できる
3. 描画中に消しゴム、アンドゥ、太さ変更、色変更、透明度変更、ペン種類変更が動作する
4. 描画モードを抜けると描画が表示されたままになり、再度フローティングボタンから描画再開できる
5. ツール終了で描画が消え、フローティング UI も閉じる

## Free Release Done Criteria By Role

### Product Owner

- 無料版で検証したい仮説が明文化されている
- ファーストリリースに含めない機能が凍結されている
- premium 候補を後から評価するための観測項目が定義されている

### Android Lead

- 主要導線で blocker 級不具合がない
- 無料版に不要な Billing 依存や購入状態管理が入っていない
- premium 候補の内部実装がある場合でも feature flag などで安全に遮断されている

### UX Designer

- 無料版単体で完結した体験になっている
- ユーザーに「使えないボタン」や「未開放タブ」を見せない
- 権限説明と終了導線が自然である

### QA Lead

- 無料版の受け入れ試験と回帰試験が定義されている
- 非公開機能の誤露出チェックが含まれている
- 実機確認対象と既知不具合が整理されている

### Security and Policy Owner

- 無料版が収益化や誤認を想起させる公開表現を持たない
- ストア説明、権限理由、サポート情報が現実運用と一致している
- 有料版検討は別判断としてログ管理されている

## Premium Prioritization Guidance

優先候補は次を満たすものから検討する。

- 無料版の基幹体験を壊さない
- overlay 描画という主価値の延長にある
- Billing なしでも内部実装を安全に切り離せる
- QA コストが爆発しない
- ストア説明や公開情報の追加負荷が比較的低い

## Premium Guardrail For Core Utility

- 無料版でも「実用に足る色数」と「基本的な描き心地」は必ず提供する
- 無料版の基本色は黒、赤、青、黄の 4 色を下限とする
- premium は無料版を不便にするためではなく、表現幅や効率を広げる方向で設計する
- 色数制限やペン種別制限を行う場合でも、無料版が体験版のように見え過ぎない線を守る

## Explicit Out Of Scope For V1

- 描画内容の保存と復元
- スクリーンショット保存
- 共有機能
- アプリ内課金
- 有料機能の購入導線
- 図形補正
- 文字入力ツール
- レイヤー管理
- 複数ページ管理
- PC 連携
- AccessibilityService 前提の特殊モード
