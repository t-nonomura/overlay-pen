# Kickoff Log 2026-03-31

## Context

- `overlay-pen` は新規立ち上げで、まだ Android 実装は存在しない
- 今回は discovery の初回ラウンドとして、チーム体制、V1 要件、実現性を整理した

## Decisions

1. プロジェクトは Android first で進める
2. V1 の主要方式は Kotlin + Foreground Service + `TYPE_APPLICATION_OVERLAY` とする
3. AccessibilityService は主方式にしない
4. V1 の推奨対応 OS は Android 12 以上とする
5. 描画終了後の passive overlay 仕様は technical spike で確定する

## Why

- オーバーレイの主機能は Android 標準の `TYPE_APPLICATION_OVERLAY` で実装可能
- ただし Android 12 以降は untrusted touch 制約があり、要件をそのまま満額で実現できるとは限らない
- AccessibilityService は Play policy と説明責任が重く、V1 の主方式には向かない

## Key Risks

- Passive Annotation 中のタッチスルーと描画不透明度の両立が難しい
- 一部アプリが `HIDE_OVERLAY_WINDOWS` により overlay を隠す
- `SYSTEM_ALERT_WINDOW` と Foreground Service に関する公開審査の説明負荷がある

## Next Actions

1. Android 技術スパイクを実施する
2. Passive Annotation の仕様を固定する
3. 権限オンボーディングの UX を設計する
4. Google Play 前提か限定配布前提かを決める
