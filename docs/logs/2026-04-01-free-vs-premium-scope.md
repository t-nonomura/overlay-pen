# Free vs Premium Scope Log 2026-04-01

## Context

- ファーストリリースは無料版のみで公開する方針を採用した
- Billing や購入 UI は初回公開に含めない
- 次の論点は、無料版の完成条件と将来 premium 候補の切り分けである

## Decisions

1. 無料版の完成条件は「基幹オーバーレイ描画体験を安定提供できること」とする
2. 無料版では、開始、描画、保持、非表示、再開、停止の主導線を優先する
3. 無料版の基本色は黒、赤、青、黄の 4 色とする
4. premium 候補はリスト化するが、V1 公開版では導線を持たせない
5. premium 候補を先行実装する場合も、無料版から安全に分離できるものに限定する
6. 進行中の判断は 5 ロール視点で評価する

## Free Version Success Signals

- 描画開始まで迷わない
- 他アプリ上での描画体験が安定している
- `Keep` と `Hide` の価値が伝わる
- レビューや問い合わせでコア体験への需要が確認できる

## Premium Screening Questions

- その機能は無料版の延長として自然か
- その機能のために無料版の品質や複雑性を損なわないか
- 無料版のままでも満足できる体験を残せるか
- 将来課金導線を入れても押し売りに見えないか
- 追加運用コストに見合う売上期待があるか

## Initial Premium Candidates

- 追加カラーパレット
- 追加ペン種類
- 描画の保存と復元
- スクリーンショット保存
- 共有機能
- 図形補正
- テキスト入力
- レイヤー管理

## Guardrails

- Billing、価格表示、購入ボタン、未解放ボタンは無料版に含めない
- premium 候補の内部実装は、公開 UI から参照できないことを前提とする
- premium 候補より無料版の安定化を優先する
- 無料版の色数やペン種類は、意図的に不便すぎる状態へ落とさない
- 黄色を無料版に含め、強調用途を premium 専用にしない
