# Release Strategy Log 2026-04-01

## Context

- `overlay-pen` は Android overlay drawing を主機能とする
- April 1, 2026 時点では、基本フローを持つプロトタイプ実装が存在する
- 今後の公開方針として、無料版を先行し、市場需要を見てから有料版を判断したい
- 現在の前提は個人デベロッパアカウント運用であり、公開情報と運用負荷の制約がある

## Decisions

1. ファーストリリースは最小限機能の無料版とする
2. 初回公開ではアプリ内課金、機能解除、購入導線を提供しない
3. 基幹機能の需要が確認できた場合のみ、有料版を次段階として検討する
4. 有料版を正式に進める場合は、個人アカウントのまま進める前提を置かず、組織アカウント運用または新規組織アカウントへのアプリ移管と、公開住所の扱いを再設計する
5. リッチ機能を先行実装する場合は、無料版で導線を開かず、内部 flag あるいは非公開設定で隔離する

## Why

- 初回公開の目的は、基幹体験の市場需要と継続利用を確認することにある
- 無料版に絞ることで、価格設計、購入導線、サポート、課金運用の複雑性を先送りできる
- Google Play の公開情報要件と payments profile の制約を踏まえると、課金導入はアカウント運用方針とセットで判断する必要がある
- 先に premium 候補機能の技術的実現性を進めておくことで、需要確認後の拡張速度を上げられる

## Risks

- premium 候補機能を先に作り過ぎると、無料版の品質向上より先に実装負債が増える
- UI から見えなくても、未解放機能が無料版の安定性やパフォーマンスへ影響する可能性がある
- 組織アカウント移行や住所公開の運用が想定より重い場合、有料版計画が遅延する

## Guardrails

- 先行実装する premium 候補は、コア体験と密接に結びつくものに限定する
- 無料版の動作確認、実機検証、ストア公開準備を premium 実装より優先する
- 非公開機能は UI、設定、通知、デバッグ導線から誤って露出しないようにする
- 有料版判断の前に、無料版の利用指標、レビュー、問い合わせ内容を整理する

## Next Actions

1. V1 無料版の完成条件を明文化する
2. 将来 premium 候補機能の一覧を作り、先行実装対象と保留対象を分ける
3. feature flag の管理方針を設計する
4. 無料版公開後に確認したい需要シグナルを定義する
